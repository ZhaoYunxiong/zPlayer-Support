## 📌 欢迎！请阅读反馈指引 

感谢您使用 zPlayer！本仓库专门用于收集用户对 zPlayer 播放器的 **Bug 报告**、**功能请求** 以及 **优化建议**。

您的反馈是改进 zPlayer 质量的关键动力。

### 1. 🔍 提交 Issue 前：先搜索

在提交新的 Issue 之前，请务必先在 **“Issues”** 列表中搜索，确认您的问题或建议是否已被他人提出。

> [!TIP]
> 如果找到相同的 Bug 或功能请求，请在原 Issue 下方补充您的信息或点击 👍 按钮表示支持。

### 2. 📝 如何提交反馈 

请务必使用我们提供的 **Issue 模板** 来提交反馈。这能确保您提供所有必要信息，从而大大提高问题被解决的效率。

点击 **Issues 选项卡**，并选择以下其中一个模板开始：

| 模板名称 | 目的 | 关注重点 |
| :--- | :--- | :--- |
| **🐛 Bug 报告 (Bug Report)** | 报告程序错误、崩溃或非预期行为。 | **复现步骤**、**版本号** |
| **✨ 功能请求 (Feature Request)** | 建议添加新功能或改进现有功能。 | **期望解决的问题** |

---

## ⚙️ Bug 报告的必填核心信息

> [!WARNING]
> 请注意：缺乏清晰**复现步骤**、**版本号** 或关键环境信息的 Bug 报告，将可能被标记为“信息不足”并被关闭，直到您补充完整。

您的 Bug 报告中需要包含以下信息：

* **zPlayer 版本号：** 请填写您使用的具体版本号。
* **操作系统：** Windows / macOS / Linux / Android / iOS。
* **处理器架构：** x64 或 ARM64。
* **复现步骤 (Steps to Reproduce)：** 描述如何一步步触发该 Bug。
